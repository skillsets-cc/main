### Step 2: Create skillset.yaml

The manifest file describes your skillset and makes it discoverable.

**Template**:
```yaml
schema_version: "1.0"

# Identity
name: "Your_Skillset_Name"
version: "1.0.0"
description: "Brief description of your skillset (10-200 characters)"

author:
  handle: "@your-github-username"
  url: "https://github.com/your-github-username"

# Verification
verification:
  production_url: "https://your-production-deployment.com"
  production_proof: "./PROOF.md"  # Optional
  audit_report: "./AUDIT_REPORT.md"

# Discovery
tags:
  - "tag1"
  - "tag2"
  - "tag3"

compatibility:
  claude_code_version: ">=1.0.0"
  languages:
    - "python"
    - "typescript"

# Lifecycle
status: "active"

# Content
entry_point: "./content/CLAUDE.md"
```

**Field Requirements**:
- `name`: Alphanumeric, hyphens, underscores only. No spaces.
- `version`: Semantic versioning (e.g., 1.0.0)
- `description`: 10-200 characters
- `author.handle`: Must match your GitHub username (e.g., @yourname)
- `tags`: 1-10 tags, lowercase, hyphens only
- `status`: One of: `active`, `deprecated`, `archived`

### Step 3: Write README.md

Your README should include:

1. **Overview**: What does this skillset provide?
2. **Installation**: How to install (CLI and manual methods)
3. **Usage**: How to use the skills and agents
4. **Examples**: Concrete usage examples
5. **Configuration**: Any customization options
6. **Requirements**: Prerequisites and compatibility
7. **License**: How others can use your skillset

**See**: [`skillsets/@supercollectible/The_Skillset/README.md`](./skillsets/@supercollectible/The_Skillset/README.md) for a complete example.

### Step 4: Download and Run the Audit Skill

Visit [skillsets.cc/contribute](https://skillsets.cc/contribute) to download the `/audit_skillset` skill.

**Installation**:
```bash
# Download the audit skill
curl -O https://skillsets.cc/tools/audit_skillset.tar.gz
tar -xzf audit_skillset.tar.gz

# Copy to your Claude Code skills directory
cp -r audit_skillset ~/.claude/skills/
```

**Run the audit**:
```bash
# Navigate to your skillset directory
cd path/to/your-skillset/

# Run the audit skill in Claude Code
/audit_skillset
```

The audit will:
- Validate `skillset.yaml` against JSON Schema
- Check required files exist
- Verify content structure
- Detect large files (>1MB)
- Scan for binary files
- Look for potential secrets
- Generate `AUDIT_REPORT.md`

**Fix any issues** reported by the audit before submitting.

### Step 6: Submit Pull Request

1. **Fork** the [skillsets-cc/registry](https://github.com/skillsets-cc/registry) repository

2. **Clone** your fork:
   ```bash
   git clone https://github.com/YOUR-USERNAME/registry.git
   cd registry
   ```

3. **Create** your skillset directory:
   ```bash
   mkdir -p skillsets/@your-username/Your_Skillset_Name
   ```

4. **Copy** your prepared files:
   ```bash
   cp -r /path/to/your-skillset/* skillsets/@your-username/Your_Skillset_Name/
   ```

5. **Commit** your changes:
   ```bash
   git add skillsets/@your-username/
   git commit -m "Add @your-username/Your_Skillset_Name

   Brief description of the skillset and what it provides.

   Production URL: https://your-production-url.com"
   ```

6. **Push** to your fork:
   ```bash
   git push origin main
   ```

7. **Open Pull Request** on GitHub with:
   - Clear title: "Add @your-username/Your_Skillset_Name"
   - Description explaining what your skillset does
   - Link to production deployment
   - Summary of production proof
   - Any justifications for warnings (e.g., binary files)

## Submission Checklist

Before submitting, verify:

- [ ] `skillset.yaml` exists and passes validation
- [ ] `README.md` has installation, usage, and examples
- [ ] `AUDIT_REPORT.md` generated and shows PASSED
- [ ] `content/` contains `.claude/` directory or `CLAUDE.md`
- [ ] Skills have clear documentation
- [ ] Agents have clear documentation
- [ ] Production proof provided (in `PROOF.md` or linked)
- [ ] All files under 1MB
- [ ] No secrets or API keys in files
- [ ] Author handle matches your GitHub username
- [ ] Tags are lowercase and relevant
- [ ] Description is clear and concise

## Updating Your Skillset

To update an existing skillset:

1. Fork the registry (if you haven't already)
2. Update your skillset files in `skillsets/@your-username/Your_Skillset_Name/`
3. Increment the version in `skillset.yaml` (follow semver)
4. Update `AUDIT_REPORT.md` (re-run `/audit_skillset`)
5. Open PR with description of changes
6. Maintainer reviews and merges

**Note**: Only the original author can update their skillset. PRs from other users will be rejected.

## Deprecating Your Skillset

To deprecate:
1. Update `status: "deprecated"` in `skillset.yaml`
2. Add deprecation notice to README
3. Optionally point users to replacement skillset
4. Open PR

To archive (remove from discovery):
1. Update `status: "archived"` in `skillset.yaml`
2. Open PR
